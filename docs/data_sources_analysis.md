# Анализ источников данных и API стратегии (Мультиспорт)

Этот документ описывает текущую архитектуру сбора данных для мультиспортивной платформы (NHL, Hockey, Football, Basketball, Volleyball), утвержденную стратегию использования API и рекомендации по дальнейшей оптимизации.

## 1. Текущая архитектура и Стратегия (Current State)

Система поддерживает 5 видов спорта и использует гибридную модель сбора данных с жестким контролем лимитов API.

### Поддерживаемые виды спорта (из `src/sports_config.py`)
*   **Hockey**: NHL, KHL, SHL, Liiga, DEL
*   **Football**: EPL, La Liga, Bundesliga, Serie A, Ligue 1
*   **Basketball**: NBA, EuroLeague, VTB, ACB, BBL
*   **Volleyball**: Superliga, Serie A, PlusLiga, Bundesliga
*   **Tennis**: (В разработке)

### Источники данных

#### А. Статистика и Результаты
1.  **NHL Edge API (`api-web.nhle.com`)**
    *   **Статус:** ✅ **Основной для NHL**.
    *   **Тип:** Официальный бесплатный API.
    *   **Использование:** Полная статистика матчей, составы, play-by-play.

2.  **FlashLive API (RapidAPI)**
    *   **Статус:** ✅ **Основной для остальных видов спорта**.
    *   **Реализация:** `src/flashlive_loader.py`.
    *   **Покрытие:** Футбол, Баскетбол, Волейбол, Европейский хоккей.
    *   **Кэширование:** Реализовано внутреннее кэширование запросов.

#### Б. Коэффициенты (Odds)
1.  **API-Sports (Hockey)**
    *   **Реализация:** `src/apisports_odds_loader.py`.
    *   **Лимиты:** 100 запросов/день (Free Tier).
    *   **Стратегия:**
        *   Жесткий лимит: **95 запросов в день** (5 оставлено на ручные тесты).
        *   Кэш: **2 часа** (7200 сек).
        *   Логика: 1 запрос на лигу в день для списка матчей + точечные запросы кэфов.

2.  **The Odds API**
    *   **Реализация:** `app.py` / `src/multi_league_loader.py`.
    *   **Лимиты:** 500 запросов/месяц.
    *   **Стратегия:** Используется как резервный источник или для специфических букмекеров (US).

3.  **AllBestBets**
    *   **Статус:** ❌ **Отказались / Deprecated**.
    *   **Действие:** Код (`src/allbestbets_loader.py`) подлежит удалению или архивации.

## 2. Утвержденная частота обновлений (API Policy)

В коде (`src/odds_monitor.py` и лоадеры) зафиксированы следующие правила для экономии ресурсов:

1.  **Автоматический мониторинг (AutoMonitor)**
    *   **Интервал:** Раз в **12 часов** (43200 сек).
    *   **Цель:** Минимизация вызовов API в фоновом режиме. Система не "долбит" API каждые 5 минут.

2.  **Политика Кэширования**
    *   **Odds Cache:** 2 часа (7200 сек) для API-Sports.
    *   **General Cache:** 5 минут (300 сек) для горячих данных в `app.py`.
    *   **Historical Data:** Обновление раз в сутки (24ч).

3.  **Обработка лимитов**
    *   Лоадеры проверяют заголовки ответов (например, `x-ratelimit-requests-remaining`) и останавливают работу **до** исчерпания лимита.

## 3. Анализ альтернатив (с учетом мультиспорта)

Т.к. мы расширились на футбол и баскетбол, требования к данным выросли.

| Сервис | Тип | Вердикт для нас |
| :--- | :--- | :--- |
| **FlashLive (RapidAPI)** | Агрегатор | **Оставить**. Хорошее покрытие всех нужных лиг (EPL, NBA и т.д.) в одном месте. |
| **API-Football / Basketball** | Агрегатор (API-Sports) | Хорошая альтернатива, если FlashLive станет дорогим. У них отдельные подписки на каждый спорт. |
| **OddsPortal Scraper** | Скрапинг | **Рекомендуется внедрить** для получения *исторических* кэфов (для ML обучения), т.к. API обычно дают историю только за деньги. |
| **Official League APIs** | Официальные | Для NBA/EPL официальные API часто закрыты или очень дороги. FlashLive/API-Sports — лучший выбор. |

## 4. План действий

1.  **Cleanup:** Удалить код `AllBestBets`, так как сервис официально признан "отказным".
2.  **Scraping:** Рассмотреть внедрение скрапера OddsPortal исключительно для накопления **исторической базы** коэффициентов (для улучшения точность ML моделей), не затрагивая оперативный мониторинг.
3.  **Monitoring:** Текущая стратегия (12ч интервал + 2ч кэш) является оптимальной для бесплатных тарифов. Изменять её не требуется.
